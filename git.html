Що таке GIT.

Git - це як залізні дороги, ти поченаєш її з команд git init(робимо проект залізної дороги), git clone URL - вказуємо куди ми будемо зберігати весь проект.

Уяви що в тебе є головна магістраль (зазвичай це гілка main аба master) де в тебе їде нормальний потяг. Ти можеш створити нову гілку (дублікат твого потяга) git checkout -b new_train, щоб спробувати встановити на потяг якусь нову деталь.

Якщо ця деталь зробить твій потяг краще. Ти можеш зберегти всі зміни в інтернеті. git add . - додає всі деталі в потяг, git commit -m "New window" - ти описуєш що встановив нові вікна в потяг, git push origin new_train - зберігаєш зміни в інтернеті. Потім ти ідеш в веб інтерфейс і там створюєш пул реквест.

Пул реквест - це процедура яка поєднує твої гілки в одне ціле. Після завершення цієї процедури всі зміни внесені в гілці вливаються в основну магістраль.

Якщо ти створив нову гілку git checkout -b angle_wheels і деталь призвела до того, що потяг став погано їхати ти можеш залишити все в цій гілці (git add. (Enter) git commit -m "Bad soluion" (Enter)) і повернутися на свою магістраль git checkout main.

Інколи хтось інший може створювати гілки і вносити зміни в своїй гілці під назвою - new_branch. Для того щоб тобі перейти до їхної гілки ти повинен створити її локально на своєму компютері git checkout -b new_branch, і завантажити її з інтернету до себе на компютер git pull origin new_branch

Цього поки вистачить. Все інше вчи і читай сам. Забудь про відео уроки. Вчися правильно гуглити. І уважно читай що пише консоль. Ти казав, що любиш все перепровірити, а перевірити те що писала тобі консоль не міг. Із-за того на завдання яке можна виконати за 5 хв., ми витратили 4 години.

Успіхів!

  • git init - вводиш цю команду 1 раз коли створюєш проект в новій папці
  • git clone URL - завантажити поект з того URL який ти вкажеш. URL - проекту можеш скопіювати на кнопці code в репозиторію
  • git status - показує статус файлів в поточній гілці. Що було змінено, додано в коміт і т.д.
  • git add . - команда яка додає всі зміни в локальний пакет гіта. "Вкладаємо лист в конверт"
  • git commit -m "Commit" - створюється коміт. "Запаковуємо конверт"
  • git push origin main - відправляємо зміни в інтернет. "Вказуємо адресу куди (origin - це означає в інтернет. main - це назва гілки в якій ти знаходишся) і відправляємо push". Це треба для того щоб хтось побачив твій код
  • git pull origin main - скачуємо зміни з інтернету. "Вказуємо адресу звідки (origin - це означає в інтернет. main - це назва гілки в якій ти знаходишся) і качаємо pull". Це треба щоб ти завантажив код собі на компютер.
  • git checkout -b BRANCH_NAME - команда яка створює нову гілку під назвою BRANCH_NAME і автоматично переходить на неї.
  • git checkout main - команда яка змінює гілки. Після виконання цієї команди ти перейдеш на гілку main. Для того щоб змінити гілки треба додати всі зміни, створити коміт і запушити в інтернет (git add. (Enter) git commit -m "Commit message" (Enter) git push origin BRANCH_NAME (Enter))
  • git branch - показує весь список локальних гілок
